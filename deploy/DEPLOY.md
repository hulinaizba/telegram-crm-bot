# Деплой RoboCompanion v2 на VPS (Zomro, Ubuntu 22.04/24.04)

Полная инструкция «с нуля»: заказ сервера на Zomro → подключение с Mac →
перенос бота → автозапуск. Ничего программировать не нужно — только
копировать команды в Terminal по порядку.

Боту хватает самого дешёвого тарифа: 1 vCPU, 512 МБ–1 ГБ RAM, любой SSD.
Домен и открытые порты не нужны — бот работает через long polling
(сам стучится в Telegram, а не ждёт входящих подключений).

## Часть 0. Заказ сервера на Zomro

1. Зайдите на **zomro.com**, зарегистрируйтесь (email + телефон, придёт код
   подтверждения по СМС).
2. В разделе **VPS/VDS-хостинг** выберите **самый дешёвый тариф**
   (обычно называется что-то вроде «VPS Micro» / «Старт») — этого с запасом
   хватит для одного Telegram-бота.
3. При выборе операционной системы обязательно возьмите
   **Ubuntu 22.04 LTS x64** или **24.04 LTS x64**.
   ⚠️ Не выбирайте Windows Server — некоторые общие гайды в интернете
   советуют его, но нашему боту он не нужен и стоит дороже.
4. Если будет выбор **панели управления** (ISPmanager и т.п.) — можно
   смело выбрать **«Без панели»**: она платная и не понадобится, весь
   перенос бота ниже делается через SSH-команды.
5. Локацию сервера выбирайте любую ближайшую по цене/скорости — на работу
   бота она не влияет.
6. Оформите заказ и оплатите (обычно доступна карта Visa/MasterCard).
7. Подождите **5–15 минут**, пока сервер активируется. После этого на
   вашу почту придёт письмо с тремя вещами:
   - **IP-адрес сервера** (например, `185.xx.xx.xx`)
   - **логин**: `root`
   - **пароль**
   
   Те же данные всегда доступны в личном кабинете Zomro:
   *Товары/Услуги → Виртуальные серверы → ваш сервер*.

Сохраните IP и пароль — они понадобятся в следующей части.

## Часть 1. Подключение к серверу с Mac

1. Откройте **Terminal** на Mac (Программы → Утилиты → Terminal, или через
   Spotlight: ⌘+Пробел → «Terminal»).
2. Введите (замените `IP_СЕРВЕРА` на ваш адрес из письма):
   ```bash
   ssh root@IP_СЕРВЕРА
   ```
3. При первом подключении спросит что-то вроде
   `Are you sure you want to continue connecting (yes/no)?` — введите `yes` и
   нажмите Enter.
4. Введите пароль root из письма (при вводе пароля в Terminal символы **не
   отображаются** — это нормально, просто напечатайте и нажмите Enter).
5. Если увидели приглашение вида `root@сервер:~#` — вы внутри сервера, всё
   получилось.

Рекомендуется сразу сменить пароль на свой:
```bash
passwd
```
(введёт запросит новый пароль дважды).

## Часть 2. Остановите бота на Mac

Прежде чем запускать бота на сервере, остановите его на своём компьютере —
Telegram не разрешает двум процессам одновременно работать с одним токеном
бота.

В терминале, где на Mac запущен `bot.py`, нажмите **Ctrl+C**.

## Часть 3. Копирование проекта на сервер

Откройте **второе окно Terminal на Mac** (первое остаётся подключённым к
серверу) и из папки проекта выполните (замените `IP_СЕРВЕРА`):

```bash
cd /Users/user/Desktop/robo_companion_v2
ssh root@IP_СЕРВЕРА "mkdir -p /opt/robocompanion"
scp bot.py config.py repository.py sheets.py stages.py templates.py \
    requirements.txt .env robo-companion-*.json \
    root@IP_СЕРВЕРА:/opt/robocompanion/
```

Каждый файл спросит пароль root (или не спросит, если уже вводили в этой
сессии) — это нормально, `scp` просто копирует файлы по защищённому каналу.

## Часть 4. Установка окружения (уже в окне, подключённом к серверу)

```bash
apt update && apt install -y python3-venv python3-pip
cd /opt/robocompanion
python3 -m venv venv
venv/bin/pip install -r requirements.txt
```

Проверьте путь к ключу Google в `.env` — откройте файл на сервере:
```bash
cat .env
```
Убедитесь, что строка `GOOGLE_CREDENTIALS_FILE=...` указывает просто на
имя файла (например `robo-companion-0c315fd18ea0.json`) без лишних папок —
он лежит рядом, в `/opt/robocompanion/`.

## Часть 5. Пробный запуск

```bash
venv/bin/python bot.py
```

Если в консоли появилось `Загружено N клиентов` и нет ошибок — всё работает.
Проверьте в Telegram команду `/today` — бот должен ответить.

Останавливаем пробный запуск: **Ctrl+C**.

## Часть 6. Автозапуск через systemd (чтобы бот жил всегда, а не только пока открыт Terminal)

```bash
cp /opt/robocompanion/deploy/robocompanion.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable --now robocompanion
```

Если папку `deploy/` не копировали — создайте файл вручную:
```bash
nano /etc/systemd/system/robocompanion.service
```
и вставьте содержимое из `deploy/robocompanion.service` (см. ниже в этом
проекте), затем `Ctrl+O`, Enter, `Ctrl+X` для сохранения и выхода.

Проверка, что всё поднялось:
```bash
systemctl status robocompanion
```
Должно быть зелёное `active (running)`.

Теперь можно закрывать оба окна Terminal и даже выключать Mac —
бот продолжит работать на сервере 24/7.

## Часть 7. Повседневное управление

```bash
systemctl status robocompanion    # состояние (работает/упал)
systemctl restart robocompanion   # перезапуск (например, после /reload не помогло)
journalctl -u robocompanion -f    # живые логи systemd (Ctrl+C для выхода)
tail -f /opt/robocompanion/bot.log  # живые логи самого бота
```

Бот сам поднимается после падения (`Restart=always` в конфиге) и после
перезагрузки сервера — руками включать ничего не нужно.

## Часть 8. Обновление кода в будущем

Когда мы будем что-то менять в коде — с Mac:

```bash
cd /Users/user/Desktop/robo_companion_v2
scp bot.py config.py repository.py sheets.py stages.py templates.py \
    root@IP_СЕРВЕРА:/opt/robocompanion/
ssh root@IP_СЕРВЕРА "systemctl restart robocompanion"
```

Две команды — и обновлённый бот уже работает на сервере.
